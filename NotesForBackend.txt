15. *** Note about MongoDB vs db.json IDs ***
    - MongoDB/Mongoose returns `_id` by default.
    - If frontend uses `item.id`, add `toJSON: { virtuals: true }` in the model so JSON response also includes `id`.
    - json-server already provides normal `id`, so `virtuals: true` is not needed there.
    - Frontend ID comparison must match the ID type:
      - json-server: `food.id === Number(itemId)`
      - MongoDB with virtual `id`: `food.id === itemId`

    Files check:
    - StoreContext.jsx
    - productModel.js
    - If another model later goes to frontend and also uses id, 
    then that model may need the same virtuals: true setup too
